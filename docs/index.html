Delet Backend

Overview

This repository hosts the backend services for the Delet platform, focusing on comprehensive APIs for property management, lock operations, user authentication, booking management, and hardware interaction.

API Documentation

To fully explore the capabilities of our APIs, please refer to the following SwaggerHub documentation:

  • Property Management API Documentation
  • SwitchBot Lock API Documentation

Directory Structure Overview

The project is structured into several directories, each encapsulating specific aspects of application functionality to promote modularity and scalability:

  • controllers/ - Handles the logic for API endpoints. Includes operations for authentication, cameras, contacts, dashboard, invoices, notifications, orders, properties, schedules, shipments, transactions, and more.
  • helpers/ - Provides utility modules and functions for common tasks. Includes helpers for account management, AWS S3 integration, camera operations, Cognito integration, email generation, and other utilities.
  • models/ - Defines the Mongoose schemas and models for the application's data structure. Includes models for accounts, actions, buildings, confirmation codes, contacts, discounts, hardware, invoices, meetings, orders, properties, shipments, transactions, users, and more.
  • routes/ - Manages all routing for the application. Defines endpoints for managing accounts, agents, cameras, contacts, dashboards, documents, hardware, invoices, meetings, notifications, orders, products, properties, shipments, subscriptions, transactions, and user interactions.

Key Modules

1. `property`

Focuses on property-related operations such as viewing, updating property details, pinging properties to check connection status, booking management, and session management. Uses authentication middleware and integrates with models for properties, hardware, and meetings.
